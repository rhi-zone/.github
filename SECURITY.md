# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any Rhizome project, please report it by opening a private security advisory in the relevant repository.

1. Go to the repository's Security tab
2. Click "Report a vulnerability"
3. Provide details about the issue

We will respond within 48 hours and work with you to understand and address the issue.

## Scope

This policy applies to all repositories under the rhi-zone organization:

- [Moss](https://github.com/rhi-zone/moss) - Structural code intelligence
- [Hypha](https://github.com/rhi-zone/hypha) - Federation protocol for persistent worlds
- [Resin](https://github.com/rhi-zone/resin) - Constructive media generation
- [Frond](https://github.com/rhi-zone/frond) - Game design primitives library
- [Dew](https://github.com/rhi-zone/dew) - Minimal expression language
- [Liana](https://github.com/rhi-zone/liana) - API bindings IR and codegen
- [Cambium](https://github.com/rhi-zone/cambium) - Pipeline orchestrator for data conversion
- [Canopy](https://github.com/rhi-zone/canopy) - Universal UI client with control plane
- [Siphon](https://github.com/rhi-zone/siphon) - Legacy software lifting framework
- [Nursery](https://github.com/rhi-zone/nursery) - Ecosystem orchestrator
- [Spore](https://github.com/rhi-zone/spore) - Lua runtime with plugin system
- [Pith](https://github.com/rhi-zone/pith) - Standard library interfaces
- [Flora](https://github.com/rhi-zone/flora) - Lua-based tools, scaffolds, and orchestration

## Supported Versions

We provide security updates for the latest release of each project. Older versions may not receive patches.
